# Planner — MVP застосунок для планування

Мінімалістичний застосунок-планер, який поєднує список задач та календарний вигляд.

## Технологічний стек

- **React 18** + **TypeScript** — UI та типізація
- **Vite** — збірка та dev-сервер
- **Dexie.js** — обгортка над IndexedDB для локального зберігання
- **date-fns** — робота з датами

## Архітектура

### Модель даних

**Task** містить:
- `id`, `title`, `is_done`
- `created_at`, `updated_at`
- `scheduled_date`, `scheduled_time` (опційно)
- `deadline_date` (опційно)
- `reminder_enabled` (тільки якщо є scheduled_date)
- `notes` (опційно)

### Зберігання даних

Дані зберігаються локально в **IndexedDB** через Dexie.js. Всі операції виконуються на клієнті, без сервера.

## Як запустити

1. Встановити залежності:
```bash
npm install
```

2. Запустити dev-сервер:
```bash
npm run dev
```

3. Відкрити браузер за адресою, яку покаже Vite (зазвичай `http://localhost:5173`)

4. Зібрати для продакшену:
```bash
npm run build
```

## Деплой на GitHub Pages

1. Закомітьте і запуште в `main` або `master`.
2. У GitHub зайдіть в **Settings → Pages** і виберіть **Source: GitHub Actions**.
3. Workflow `.github/workflows/deploy.yml` автоматично збере і задеплоїть `dist`.

Якщо репозиторій має іншу основну гілку — замініть її у `deploy.yml`.

## Функціональність MVP

### ✅ Реалізовано

1. **Створення задач** з підтримкою:
   - Задач без дати (беклог)
   - Задач на конкретну дату (з опційним часом)
   - Задач з дедлайном

2. **Екран "Сьогодні"** з 4 секціями в точному порядку:
   - Задачі на цей день (без часу) — сортування за `created_at` ASC
   - Задачі на цей день (з часом) — сортування за часом
   - Задачі з дедлайном на цей день — сортування за `created_at` ASC
   - Задачі без дати (беклог) — сортування за `created_at` ASC

3. **Календарний вигляд** (місяць):
   - Відображення задач з `scheduled_date` та `deadline_date`
   - Візуальне розрізнення типів задач
   - Клік по дню відкриває екран "Сьогодні" для обраної дати

4. **CRUD операції**:
   - Створення задач
   - Редагування задач
   - Відмітка як виконана/невиконана
   - Видалення задач

5. **Додаткові фічі**:
   - Перемикач "Показувати виконані"
   - Вибір дати на екрані "Сьогодні"
   - Відображення прострочених дедлайнів

### ⚠️ Обмеження MVP

- **Нагадування**: Поле `reminder_enabled` зберігається, але системні сповіщення не реалізовані (потребує додаткових API браузера)
- **Пошук**: Не реалізовано (опційно згідно з ТЗ)
- **PWA**: Не налаштовано (можна додати пізніше)

## UX рішення

### Правила для задач

- Задача може мати одночасно `scheduled_date` і `deadline_date` (MVP дозволяє це)
- Якщо задача має `scheduled_time` і користувач змінює дату — час залишається, нагадування переноситься
- Прострочені дедлайни показуються з маркером "(прострочено)" без агресивних банерів
- Використовується локальний час системи

### Дизайн

- Мінімалістичний чорно-білий стиль
- Без зайвих декоративних елементів
- Чиста типографіка
- Багато повітря (whitespace)

## Що далі (out of scope для MVP)

- Командна робота, шерінг
- Проєкти/папки/теги
- Повторювані задачі
- Складні фільтри, аналітика
- Інтеграції з календарями
- Вкладені підзадачі
- Системні нагадування (потребує додаткових дозволів)

## Розробка

Проект використовує TypeScript у строгому режимі. Всі компоненти написані з урахуванням доступності (базові ARIA атрибути, клавіатурна навігація).

Для локальної розробки дані зберігаються в IndexedDB браузера. Щоб очистити дані, можна використати DevTools браузера (Application → IndexedDB → PlannerDB).
